# AQI Prediction using Machine Learning

A machine learning pipeline that predicts the Air Quality Index (AQI) of Indian cities from daily pollutant concentration data, comparing multiple regression models and tuning the best-performing one.

## Overview

This project (`main.ipynb`) loads a city-wise daily air quality dataset, cleans and prepares it, trains and compares several regression models to predict AQI, tunes the best model's hyperparameters, and saves the final model along with sample predictions.

## Dataset

- **File expected:** `city_day.csv`
- **Target variable:** `AQI`
- **Features:** pollutant concentration columns (e.g. PM2.5, PM10, NO2, SO2, CO, O3, etc.)
- **Dropped columns:** `City`, `Date` (categorical/non-numeric), `AQI_Bucket` (redundant with `AQI`)

Place `city_day.csv` in the same directory as the notebook before running.

## Workflow

1. **Data loading** — read `city_day.csv` into a pandas DataFrame.
2. **Data preprocessing**
   - Drop rows with a missing `AQI` value (the prediction target).
   - Fill remaining missing values in numeric columns with the column mean.
   - Drop non-numeric/irrelevant columns (`City`, `Date`, `AQI_Bucket`).
3. **Exploratory analysis** — correlation heatmap of numeric features against `AQI` (via seaborn).
4. **Train/test split** — 80/20 split (`random_state=42`).
5. **Feature scaling** — `StandardScaler` fit on training data, applied to both train and test sets.
6. **Model comparison** — trains and evaluates three regressors on MAE, RMSE, and R²:
   - Linear Regression (baseline)
   - Random Forest Regressor
   - XGBoost Regressor
7. **Hyperparameter tuning** — `GridSearchCV` (5-fold CV, scored on R²) over XGBoost's `n_estimators`, `learning_rate`, and `max_depth`.
8. **Final model training** — retrains XGBoost with the tuned parameters and evaluates it on the test set.
9. **Model persistence** — saves the trained model(s) with `joblib`:
   - `aqi_predictor_best.pkl`
   - `optimized_xgb_model.pkl`
10. **Prediction & export** — predicts AQI for sample test rows, saves them to `aqi_predictions.csv`, and plots actual vs. predicted AQI.

## Usage

1. Place `city_day.csv` in the project directory.
2. Open and run `main.ipynb` top to bottom in Jupyter.
3. Outputs generated by the notebook:
   - `aqi_predictor_best.pkl` — saved baseline XGBoost model
   - `optimized_xgb_model.pkl` — saved tuned XGBoost model
   - `aqi_predictions.csv` — predicted AQI values for sample test rows

## Output Files

| File | Description |
|---|---|
| `aqi_predictor_best.pkl` | Serialized baseline XGBoost model |
| `optimized_xgb_model.pkl` | Serialized XGBoost model after hyperparameter tuning |
| `aqi_predictions.csv` | Sample AQI predictions on held-out test data |

## Notes / Possible Improvements

- Currently drops `City` and `Date` entirely — re-encoding `City` (one-hot) and extracting date features (month, season) could improve accuracy given AQI's strong seasonality.
- Mean-imputation of missing values is simple; median or interpolation-based imputation may be more robust to outliers.
- Consider cross-validating all three models (not just XGBoost) before committing to XGBoost for tuning.
